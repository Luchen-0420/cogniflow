产品简介

CogniFlow 是一款完全开源的 AI 驱动智能信息管理工具，旨在解决现代工作流中的信息碎片化问题。它通过提供一个统一的记录入口和强大的 AI 智能处理能力（基于智谱 GLM-4），实现了“零心智负担”的记录体验。
官网链接： https://ci.ai-knowledgepoints.cn/
项目地址： https://github.com/alg-bug-engineer/cogniflow

核心竞争力在于其巧妙地将 AI 快捷输入（自然语言识别与声明式前缀）、多样化信息管理（任务、日程、笔记等五种卡片）以及 灵活的部署架构（个人本地存储与团队 PostgreSQL 数据库）相结合。

该项目不仅为个人用户提供了高效的知识和任务管理方案，也通过 Docker Compose 和 PostgreSQL 支持，为团队和企业提供了可私有化部署、支持多端同步的协作平台。

核心特性

  • AI 快捷输入是实现“零心智负担”的核心亮点
  • 功能覆盖信息记录、整理、追踪、分析的全周期
  • 灵活的存储模式兼顾个人隐私与团队协作需求
  • MIT 协议与 Docker 部署使其具备高商用价值

产品亮点

  • AI 驱动的自然语言输入
  • 五合一卡片（任务、日程、笔记等）统一管理
  • 支持本地或数据库（团队同步）部署

功能设计

CogniFlow 的核心亮点在于其AI 驱动的快捷输入机制。它允许用户像聊天一样输入信息，AI 自动识别意图并创建相应卡片，彻底消除了传统工具中“先思考分类，再填写表单”的繁琐流程[1]。

示例: "明天下午3点开会" → 自动创建为日程。

示例: "完成报告 #重要" → 自动创建为任务。

技术优势

结合 AI 智能与用户控制，提供了多种输入方式：

  • 自然语言输入: 基于智谱 GLM-4 的意图识别。
  • 声明式前缀: 用户可通过 `任务:`、`笔记:`、`@会议` 等前缀明确指定类型，AI 辅助提取关键信息。
  • 智能模板: 输入“日报”、“会议”等触发词，快速调用预设模板。

AI 核心

  • • AI 模型: 智谱 GLM-4
  • • 处理方式: 流式处理
  • • 配置: .env 文件配置 API Key
  • • 成本控制: 内置预算设置与日志

多样化信息管理

统一管理所有碎片化信息

核心卡片系统

CogniFlow 预设了五种智能卡片类型，覆盖了个人和团队信息管理的主要场景：

  • 📋 任务卡片: 优先级、状态、截止日期
  • 📅 日程卡片: 开始/结束时间、冲突检测
  • 📝 笔记卡片: 富文本、Markdown 支持
  • 📚 资料卡片: 附件支持、分类归档
  • 🔗 链接卡片: 自动抓取标题和图标

灵活的存储架构

兼顾个人隐私与团队协作

模式一：本地存储 (个人)

默认模式，数据存储在浏览器（IndexedDB或LocalStorage），实现完全的数据本地化和隐私安全。

  • 开箱即用，无需配置
  • 离线可用，隐私安全
  • 不支持多设备同步

模式二：数据库 (团队/多端)

通过连接 PostgreSQL 数据库，实现数据持久化存储、多设备自动同步和团队协作[1]。

  • 数据持久化，多端同步
  • 支持团队协作与数据备份
  • 支持 Docker 一键部署

部署与技术栈

项目提供 `docker-compose.yml` 及一键部署脚本 (`./deploy-all.sh`)，快速启动后端服务、PostgreSQL 数据库和 pgAdmin 管理界面。

数据库: PostgreSQL (10+ 核心表)

后端: Node.js + Express

数据可视化

CogniFlow 不止于记录，更提供了智能报告与统计功能，帮助用户分析工作状态和时间分配[1]。
