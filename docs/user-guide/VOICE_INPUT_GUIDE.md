# 🎤 语音输入功能使用说明

## 快速开始

### 1. 找到语音输入按钮

在 CogniFlow 主界面的输入框左侧，你会看到：

```
[📎] [🎤] [────────输入框────────] [发送]
       ↑
   语音输入按钮
```

### 2. 开始使用

**步骤 1**: 点击麦克风按钮 🎤

**步骤 2**: 浏览器会请求麦克风权限（首次使用）
- 点击"允许"以启用语音输入功能

**步骤 3**: 按钮变为红色并开始脉冲动画
- 这表示正在录音，请开始说话

**步骤 4**: 对着麦克风说话
- 说话内容会实时显示在浮窗中
- 识别的文字会自动添加到输入框

**步骤 5**: 再次点击按钮停止录音
- 或者说完后等待自动停止

## 最佳实践

### ✅ 推荐做法

1. **清晰说话**
   - 语速适中，不要太快或太慢
   - 吐字清晰，避免含糊不清

2. **安静环境**
   - 在相对安静的环境中使用
   - 减少背景噪音干扰

3. **合适距离**
   - 保持麦克风距离 20-50cm
   - 不要太远或太近

4. **自然表达**
   - 像平时说话一样自然表达
   - 可以说"逗号"、"句号"等标点

### ❌ 避免的情况

1. **语速过快** - 识别准确率会下降
2. **环境嘈杂** - 背景噪音会影响识别
3. **距离太远** - 声音太小可能无法识别
4. **频繁切换** - 频繁开关可能导致识别不稳定

## 常见问题

### Q1: 点击按钮没有反应？

**可能原因**:
- 浏览器不支持语音识别
- 没有授权麦克风权限
- 麦克风硬件问题

**解决方法**:
1. 确认使用的是 Chrome 或 Edge 浏览器
2. 检查浏览器设置中的麦克风权限
3. 测试麦克风是否正常工作
4. 刷新页面重试

### Q2: 识别不准确怎么办？

**改进方法**:
1. 说话更清晰，语速适中
2. 靠近麦克风，保持合适距离
3. 减少背景噪音
4. 使用质量更好的麦克风
5. 检查网络连接状态

### Q3: 如何开启麦克风权限？

**Chrome/Edge**:
1. 点击地址栏左侧的锁图标
2. 点击"网站设置"
3. 找到"麦克风"选项
4. 选择"允许"

**Safari**:
1. Safari 菜单 → 偏好设置
2. 网站 → 麦克风
3. 找到网站，选择"允许"

### Q4: 可以识别英文吗？

**回答**: 
- 目前主要支持中文
- 也能识别英文，但准确率可能稍低
- 中英文混合输入也能识别

### Q5: 会上传我的语音吗？

**隐私说明**:
- ❌ 不会上传音频文件
- ✅ 所有处理在本地完成
- ✅ 只获取文字识别结果
- ✅ 完全保护隐私安全

### Q6: 为什么 Firefox 不能用？

**原因**: 
Firefox 浏览器目前不支持 Web Speech API，这是浏览器的限制，不是软件问题。

**解决**: 
请使用 Chrome、Edge 或 Safari 浏览器。

## 使用技巧

### 技巧 1: 配合手动输入

你可以先打一些字，然后用语音补充：

```
手动输入: "会议主题："
语音输入: "讨论第三季度产品规划"
结果: "会议主题：讨论第三季度产品规划"
```

### 技巧 2: 说标点符号

说话时可以说出标点符号：

```
说: "今天完成了三个任务 逗号 明天继续推进项目 句号"
结果: "今天完成了三个任务，明天继续推进项目。"
```

### 技巧 3: 长篇输入分段

对于很长的内容，建议分段录入：

```
1. 录第一段 → 停止
2. 检查内容是否正确
3. 继续录第二段 → 停止
4. 以此类推
```

### 技巧 4: 结合模板使用

先选择模板，再用语音填充内容：

```
1. 输入 "/" 选择"会议纪要"模板
2. 点击语音输入
3. 说出会议内容
4. 自动填充到相应字段
```
